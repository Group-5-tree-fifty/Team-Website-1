Our Development Process

We follow agile software development methods to stay flexible and improve our project. It's okay if something isn't perfect the first time. Our goal is to continuously improve our work and produce code that is clean, readable, and maintanable. We all get stuck sometimes, so we encourage asking questions and supporting each other.

So how does our journey help you? With every problem/ obstacle we encounter on our software journey, we note it, solve it, document the solution, and pass on the information to other team members and then to you as new programmers!

Julio Contribution: By sharing what we learn, we help create a knowledge base that others can learn from and build upon. This allows new developers to avoid common mistakes, understand problem-solving approaches, and improve their coding skills more efficiently. Our goal is not only to complete our project, but also to contribute useful knowledge that supports learning and collaboration within the programming community.
